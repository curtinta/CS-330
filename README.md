CS‑330 Portfolio Reflection
How do I approach designing software?
I approach software design by breaking large problems into smaller, manageable components and focusing on how each part contributes to the overall user experience. I start by identifying the core features, then design classes and structures that support those features. This project reinforced the importance of planning object relationships early, especially when building interactive 3D scenes.

What new design skills did this project help me develop?
This project strengthened my ability to design modular graphics systems, including object placement, lighting, camera control, and scene composition. I also learned how to translate a 2D reference image into a fully realized 3D environment.

What design process did I follow?
My process involved analyzing the reference image, sketching object placement, building the scene step‑by‑step, and refining lighting and textures. I iterated frequently, testing each change visually before moving forward.

How can these design tactics help in future work?
Breaking tasks into small visual tests and iterating quickly is useful in any graphics, UI, or game development project. This approach reduces debugging time and improves clarity.

How do I approach developing programs?
I develop programs iteratively—starting with a basic version, then adding features and refining functionality. I rely on modular classes, consistent naming, and clear comments to keep the code maintainable.

What new development strategies did I use?
I learned to manage transformations more effectively, including scaling, rotation, and translation. I also improved my understanding of shader‑based lighting and camera movement.

How did iteration factor into development?
Iteration was essential. Each object, texture, and light source required multiple adjustments to achieve the correct look. Visual testing after each change ensured accuracy.

How has my development approach evolved?
Throughout the milestones, I became more structured in my workflow—planning ahead, organizing files, and writing cleaner, more modular code. By the final project, I was able to integrate all components smoothly.

How can computer science help me reach my goals?
Computer science provides the technical foundation to build interactive systems, solve complex problems, and create software aligned with my career goals in development and technology.

How do computational graphics support my educational pathway?
They deepen my understanding of math, physics, and rendering techniques, which will support future coursework in game development, simulations, and advanced graphics.

💼 How do computational graphics support my professional pathway?
These skills translate directly into fields like game development, UI/UX design, simulation engineering, and software visualization. Understanding how to build and optimize 3D scenes is valuable in many modern tech roles.
